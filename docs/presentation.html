VGen

Video Creative Generation Platform

Campaign-based video merging at scale. Upload intro hooks and main videos, define quality parameters, get all combinations merged automatically.

Live on GCP Cloud Run + Pub/Sub + GCS

The Problem

Without VGen

  • Marketing manager has 10 intros and 5 main videos
  • Needs 50 unique combinations for ad testing
  • Manual editing in Premiere/Final Cut: hours of work
  • Each new campaign repeats the pain

With VGen

  • Upload intros and mains to a campaign
  • Click "Launch" -- all 50 combinations generated
  • Stream-copy merge: under 10 seconds
  • Download individually or as a batch

Data Flow -- Upload to Download

1
Get signed URL
POST /api/upload/signed-url → GCS signed upload URL (60min expiry)
2
Direct upload to GCS
Browser PUTs video bytes directly to Cloud Storage. API never touches video data.
3
Register + ffprobe
POST /api/campaigns/{id}/videos → backend runs ffprobe to extract codec, resolution, duration
4
Launch → Pub/Sub fan-out
N intros x M mains = N*M messages published instantly to "merge-tasks" topic
5
Worker processes
Pull message → download from GCS → FFmpeg merge → upload to GCS → update Firestore

Worker Internals

Merge Strategy

  • Stream-copy when intro + main share codec, resolution, fps (~2s)
  • Re-encode when incompatible or user selected h264/h265 (~30-60s)
  • Compatibility checked via ffprobe before merge

Reliability

  • Idempotent: skips already-completed combinations
  • Always acks Pub/Sub messages (no infinite redelivery)
  • Failed combinations tracked in Firestore with error detail
  • Dead-letter topic after 5 retries

Why Pub/Sub (not Cloud Tasks)

Cloud Tasks caps at 500 dispatches/sec. A 100-combo campaign at 100K DAU needs instant fan-out. Pub/Sub: unlimited, built-in DLQ, lower cost.

Why GCS (not Drive)

Drive: 12K API req / 100s / project. At 100K DAU, exceeded in seconds. GCS signed URLs bypass API entirely -- browser talks direct to storage.

DLQ & Incident Response

Dead-Letter Queue Flow

  Worker fails to process message
          |
          v  (Pub/Sub retries x5)
  Still failing after 5 attempts
          |
          v
  Message → merge-tasks-dlq topic
          |
          v
  merge-tasks-dlq-sub subscription
          |
  +-------+-------+
  |               |
  v               v
  Alert email   API inspection
  sent          GET /api/dlq/messages

Inspection API

Peek at failed messages

curl /api/dlq/messages

Returns combination_id, campaign_id, error details. Messages stay in queue.

Dismiss after review

curl -X POST /api/dlq/purge/{msg_id}

Acknowledges and permanently removes the message.

CLI Alternative

gcloud pubsub subscriptions pull \
  merge-tasks-dlq-sub --limit 5
